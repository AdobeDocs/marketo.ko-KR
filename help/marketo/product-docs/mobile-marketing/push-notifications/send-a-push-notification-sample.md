# 푸시 알림 샘플 보내기 {#send-a-push-notification-sample}

샘플을 보내 푸시 알림이 올바르게 작동하는지 확인할 수 있습니다.

>[!PREREQUISITES]
>
>샘플을 전송하는 장치에 앱이 설치되어 있는지 확인합니다.

1. 다음 세 가지 방법으로 샘플을 보낼 수 있습니다.

   왼쪽의 트리에서 푸시 알림 자산을 마우스 오른쪽 단추로 클릭하고 를 클릭합니다 **샘플 보내기**.

   ![](assets/image2015-7-13-11-3a26-3a15.png)

   또는 푸시 알림 작업 메뉴에서 **샘플 보내기**.

   ![](assets/image2015-7-13-11-3a28-3a37.png)

   또는 푸시 알림 편집기에서 **샘플 보내기**.

   ![](assets/image2015-7-20-13-3a29-3a3.png)

1. 사용할 테스트 장치를 이미 설정한 경우 목록에서 해당 장치를 선택하면 됩니다.

   ![](assets/image2015-7-29-8-3a25-3a17.png)

   또는 **장치 추가** to [새 테스트 장치 추가](/help/marketo/product-docs/mobile-marketing/push-notifications/adding-a-new-test-device.md) 샘플을 로 보내려면

   ![](assets/image2015-7-13-11-3a34-3a21.png)

1. 장치를 앱에 연결하는 두 가지 방법이 있습니다.

   첫 번째 옵션을 사용하여 **장치가 있는 방문 URL** 버튼을 클릭하고 필드에서 URL을 복사하여 이메일 또는 텍스트 메시지로 장치에 보냅니다. 장치에서 URL을 누릅니다. 상태에 연결이 표시되면 을 클릭합니다. **보내기**.

   ![](assets/image2015-7-29-8-3a29-3a18.png)

   또는 두 번째 옵션을 사용하여 **장치로 QR 코드 스캔** 버튼을 클릭하고 장치를 사용하여 QR 코드를 스캔합니다. 상태에 연결이 표시되면 을 클릭합니다. **보내기**.

   ![](assets/image2015-7-29-8-3a31-3a20.png)

   >[!TIP]
   >
   >테스트 장치를 추가했지만 다음에 나열되지 않았습니다. **보내기** 필드? 문제를 해결하려면 다음을 확인하십시오.
   >
   >* 푸시 알림과 연결된 앱에 대해 푸시 알림이 켜집니다.
   >
   >* 푸시 알림은 테스트 장치의 플랫폼에 대해 구성됩니다. 예를 들어 iPhone을 테스트 장치로 추가한 경우 iOS에 대해 푸시 알림이 활성화되어 있는지 확인합니다.
