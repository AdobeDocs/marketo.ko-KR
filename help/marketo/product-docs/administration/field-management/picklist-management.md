# 선택 목록 관리 {#picklist-management}

선택 목록 관리를 사용하면 필드에 대해 고정된 값 집합을 정의하여 Marketo Engage 내의 데이터 및 워크플로우 관리를 단순화할 수 있습니다. 정의된 선택 목록이 있는 CRM 필드에 매핑되지 않은 텍스트가 아닌 필드만 Marketo에서 관리할 수 있습니다. 필드가 정의된 선택 목록이 있는 CRM 필드에 매핑되는 경우 해당 필드에 대한 값을 CRM에서 정의해야 합니다.

선택 목록의 상태는 해당 필드 관리 페이지에서 확인할 수 있습니다. 필드에는 다음 선택 목록 상태 중 하나가 있을 수 있습니다.

* **관리됨**: 사용자가 이 필드에 자동 제안할 수 있는 값 집합을 정의했습니다. 필드 관리에 정의된 값만 자동 제안됩니다. 관리되는 선택 목록이 삭제되면 선택 목록 상태는 필드의 초기 값(관리되지 않음 또는 초기 설정됨)으로 되돌아갑니다.

* **관리되지 않음**: 이 필드에 대해 가능한 값이 정의되지 않았습니다. 값은 데이터베이스의 해당 필드에 있는 값을 기반으로 자동 제안됩니다.

* **초기 설정**: 필드에 사용자에게 제안되는 시스템 정의 값 목록이 있습니다.

* **CRM**: 필드에 인스턴스와 동기화되는 CRM 시스템 Salesforce.com 또는 Microsoft Dynamics에 의해 정의된 값이 있습니다.

  ![](assets/picklist-management-1.png)

## 선택 목록 관리 {#manage-picklist}

필드 값을 수정하려면 **관리자** > **필드 관리**(으)로 이동하여 원하는 필드를 선택하십시오.

_필드 작업_ 드롭다운을 클릭하고 **선택 목록 관리**&#x200B;를 선택합니다.

![](assets/picklist-management-2.png)

_선택 목록 관리_ 대화 상자에서 값을 추가, 편집 또는 삭제할 수 있습니다. 관리되는 선택 목록을 삭제하여 필드를 원래 선택 목록 상태(_관리되지 않음_ 또는 _시드됨_)로 되돌릴 수도 있습니다.

![](assets/picklist-management-3.png)

각 선택 목록 항목에는 표시 값과 제출된 값이 있습니다. 디스플레이 값은 스마트 목록, 스마트 캠페인 또는 양식을 작성할 때 사용자에게 제안되는 값이며 제출된 값은 저장되는 값입니다. 예를 들어 지역 코드 사용 사례에서는 두 글자 코드(AB)를 저장하는 동안 지역의 전체 이름(예: Alberta)을 제안할 수 있습니다.

## 자동 제안 {#autosuggest}

### 관리되는 선택 목록과 관리되지 않는 선택 목록 간 전환 {#switching}

대부분의 Marketo Engage 구독에는 관리되는 선택 목록이 도입되기 전의 데이터가 포함됩니다. 이 관리되지 않는 버전 선택 목록의 값 또는 흐름 단계(예: 데이터베이스의 레코드에 있는 전체 값 집합에서)를 사용하려면 스마트 목록 또는 캠페인 보기에서 관리되는 선택 목록 설정을 전환하십시오. 켜면 관리되는 선택 목록 값만 표시됩니다. 해제하면 관리되지 않는 선택 목록이 사용되고 데이터베이스의 기존 값을 기반으로 값이 자동 제안됩니다.

### 양식 선택 목록(유형 필드 선택) {#form-picklists}

초기 설정 및 CRM 관리 선택 목록과 같이 관리 선택 목록의 값은 선택 필드 유형을 사용할 때 Forms으로 전파됩니다. 관리되는 선택 목록이 있는 필드의 경우 해당 필드를 선택하고 필드 유형을 _선택_(으)로 설정합니다.

![](assets/picklist-management-4.png)

해당 필드에 대해 정의된 관리되는 선택 목록 값 집합을 표시합니다.

![](assets/picklist-management-5.png)
