# 역할 권한 설명 {#descriptions-of-role-permissions}

>[!NOTE]
>
>**FYI**
>
>Marketing은 이제 모든 구독 간의 언어를 표준화하므로 구독에 리드/리드 및 docs.markto.com에 있는 사람/사람을 볼 수 있습니다. 이 용어는 같은 것을 의미한다.아티클 지침에는 영향을 주지 않습니다. 다른 변화도 있습니다 [자세한](http://docs.marketo.com/display/DOCS/Updates+to+Marketo+Terminology)내용

다음은 역할에 할당할 수 있는 사용 가능한 모든 권한 목록입니다. 권한은 일반적으로 Marketing To 내의 특정 기능 영역과 연결되며, 사용자가 액세스할 수 있는 영역과 기능을 제어하는 데 도움이 됩니다.

권한에 대한 일부 추가 정보:

* &quot;액세스&quot; 권한은 응용 프로그램의 해당 부분을 보고 편집할 수 있는 권한을 제공합니다.
* 하위 권한(&quot;만들기&quot;, &quot;삭제&quot; 등)에 액세스할 수 있는 역할에 대해 해당 역할의 해당 부분에 대한 &quot;액세스&quot; 권한이 있어야 합니다. 예를 들어 캠페인 편집 권한을 다른 사람에게 제공하려면 마케팅 활동에 액세스할 수 있는 전체 권한이 있어야 합니다.
* 사용 권한이 없는 작업 또는 자산을 볼 수 있습니다. 그러나 액세스 권한을 시도하면 제한된 액세스 권한에 대해 경고 메시지가 표시됩니다.

## 액세스 관리자  {#access-admin}

관리자의 내 계정 섹션에서 설정을 보고 변경합니다.

* 액세스 감사 추적 - 사용자에게 자산 감사 추적 및 관리 감사 추적 모두에 대한 액세스 권한 제공
* 액세스 채널 - 다른 사용자 지정 태그가 아닌 채널 태그에만 액세스할 수 있도록 합니다.
* 액세스 통신 제한 - 관리에서 통신 제한을 활성화할 수 있는 액세스 권한을 사용자에게 제공합니다.
* CRM 액세스 - 관리자가 Salesforce 또는 Microsoft Dynamics와 같은 CRM에 액세스할 수 있도록 제공
* Access [Data.com](http://Data.com) - 사용자에게 Data.com 흐름 작업에 대한 액세스 권한 제공
* 액세스 이메일 관리자 - 구독 취소 및 브랜딩 도메인과 같은 기본 설정을 변경하려면 이메일 관리자에게 알려줍니다
* 액세스 이벤트 파트너 - 관리자가 LaunchPoint에 액세스할 수 있도록 합니다.
* 액세스 필드 관리 - 관리자가 필드 관리에 액세스할 수 있도록 합니다.
* 액세스 파일 업로드 - 사용자에게 Design Studio에 이미지와 파일을 업로드할 수 있는 기능 제공
* 랜딩 페이지 액세스 - 사용자에게 관리의 랜딩 페이지에 대한 액세스 제공
* 액세스 위치 - 사용자에게 기본 언어, 로케일, 표준 시간대 및 통화를 설정하는 데 대한 액세스 권한을 관리 내에서 제공합니다
* 액세스 로그인 내역 - 사용자에게 감사 추적 내 사용자 로그인 기록에 대한 액세스 권한을 제공합니다.
* 액세스 로그인 설정 - 보안, IP 제한 및 스마트 목록 보고서 설정에 대한 관리자의 로그인 설정에 액세스할 수 있도록 합니다.
* 마케팅 사용자 지정 활동 액세스 - 관리자가 Marketing to 사용자 지정 활동에 액세스할 수 있도록 합니다.
* Marketing To 사용자 지정 개체 액세스 - 관리자가 Marketing To 사용자 지정 개체에 액세스할 수 있도록 합니다.
* Munchkin 액세스 - GIves 사용자가 추적 코드 설정, 사람 추적 및 API 구성 활성화를 위해 관리의 Munchkin에 액세스할 수 있습니다.
* 액세스 수익 주기 분석 - 동기화 요약 및 기여도 설정을 위해 관리에서 매출 주기 분석에 액세스할 수 있도록 사용자에게 제공
* 액세스 역할 - 사용자가 역할을 관리 및 편집할 수 있는 액세스 권한을 제공하지만 사용자는 그렇지 않습니다.
* Sales Insight 액세스 - Status, API Configuration, Person Scoring 및 기타 설정을 위한 Sales Insight in Admin의 관리 액세스 권한을 사용자에게 제공합니다
* SSO(Single Sign-On) 액세스 - SAML을 활성화하고 SAML 설정 및 리디렉션 페이지 URL을 사용하는 경우 관리자가 Single Sign-On을 관리할 수 있는 액세스 권한을 사용자에게 제공합니다
* 스마트 캠페인 액세스 - 자격을 갖춘 사람에 대한 제한을 제한하는 데 필요한 관리 기능의 스마트 캠페인에 액세스할 수 있도록 해줍니다
* SOAP API 액세스 - 관리자가 웹 서비스에서 SOAP API를 관리하는 데 액세스할 수 있도록 합니다.
* 액세스 태그 - 채널 태그를 제외한 모든 사용자 지정 태그에 액세스할 수 있도록 합니다.
* 액세스 보물 상자 - 관리 부문 보물상자에 있는 실험적인 기능을 이용할 수 있습니다.
* 액세스 사용자 - 관리자가 아닌 사용자를 편집 및 관리할 수 있는 액세스 권한을 사용자에게 제공
* 액세스 웹후크 - 세부 사항 및 응답 매핑을 설정하기 위해 관리에서 Webhook에 사용자를 제공합니다.
* 액세스 작업 영역 및 파티션 - 관리자가 작업 영역 및 파티션을 생성, 편집 및 삭제할 수 있는 액세스 권한을 사용자에게 제공합니다.

액세스 분석

선택 사항을 선택하지 않는 한 사용자에게 분석 탭, 이메일 인사이트, 보고서 및 아래 세 가지 항목에 대한 액세스 권한을 제공합니다.

* 수익 탐색기 액세스 - 체크 표시를 해제하면 매출 탐색기에 대한 사용자의 액세스 권한이 제거됩니다
* 보고서 삭제 - 선택 해제하면 사용자가 보고서를 삭제할 수 있는 기능이 제거됩니다
* Analytics 데이터 내보내기 - 검사 취소 - 사용자의 Analytics 데이터 내보내기 기능이 제거됩니다
